## Casos de Teste

> [!info]- Sobre
> **Casos de Teste** são cenários específicos criados para verificar se cada parte do projeto funciona como deveria. Eles descrevem exatamente quais entradas serão fornecidas, quais passos serão executados e qual resultado o sistema deve apresentar. Isso garante que cada funcionalidade seja validada de forma objetiva, rastreável e reproduzível, facilitando encontrar erros, confirmar comportamentos e manter a qualidade do software conforme o projeto evolui.

## Log de Bugs

> [!info]- Sobre
> O **Log de Bugs** é um registro organizado de todos os erros encontrados durante o desenvolvimento ou testes do projeto. Ele serve para documentar _quando_ o problema ocorreu, _qual foi o bug_, _qual o status atual_ (aberto, resolvido, em análise) e _como corrigir ou como foi corrigido_. Manter esse registro atualizado ajuda a rastrear problemas recorrentes, entender falhas do sistema, evitar que erros antigos retornem e melhorar a qualidade geral do software. É uma ferramenta essencial para qualquer processo profissional de desenvolvimento.
