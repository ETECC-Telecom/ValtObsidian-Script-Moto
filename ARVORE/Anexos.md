> [!info]- Sobre
> O campo **Anexos** é o espaço onde você reúne todos os materiais complementares que ajudam a entender, justificar ou aprofundar o projeto, mas que não fazem parte direta do código. Aqui entram arquivos como PDFs, imagens, capturas de tela, diagramas, dumps de banco, arquivos de configuração, documentos de referência, planilhas ou links externos relevantes. Esse espaço funciona como um repositório de suporte, garantindo que qualquer pessoa que revisite o projeto consiga acessar rapidamente todos os materiais auxiliares necessários sem precisar procurar em outros lugares.

### Dumps / Dados

> [!todo]- Sobre
> **Dumps** são arquivos que armazenam uma cópia dos dados do sistema em um formato exportado — por exemplo, um _dump de banco de dados_ (`.sql`), um _arquivo JSON_ com dados de teste, ou um _snapshot_ de uma API. Eles servem para restaurar, simular ou testar o comportamento do sistema com informações reais. Já a parte “**Dados**” inclui qualquer arquivo contendo dados estruturados ou semiestruturados usados durante o desenvolvimento: CSVs, JSONs, XMLs, planilhas e arquivos criados para reproduzir problemas, alimentar rotinas, treinar modelos ou validar funcionalidades.
> **📌 Exemplos diretos**
> - `dump_banco.sql` → uma cópia do banco usada para reproduzir ambiente
> - `usuarios_teste.json` → conjunto de usuários falsos para testar login
> - `tabela_precos.csv` → dados usados para testes de cálculos
> - `payload_api.json` → exemplo de resposta real de uma API
> - `erro_500_dump.txt` → dados capturados no momento de um erro
